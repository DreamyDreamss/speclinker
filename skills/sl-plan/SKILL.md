# /sl-plan — 기획서 × 스펙 교차 분석

## 역할

기획서 또는 변경요청 도착 시 RECON 스펙과 자동 교차 분석한다.  
어떤 INF/UIS/SCH가 영향받는지, 변경 규모가 어느 정도인지 파악하고 `/sl-analyze` 진입 전 방향을 잡는다.

```
호출:
  /sl-plan 기획서.md
  /sl-plan SR-2026-001.md
  /sl-plan "변경 내용 자유 텍스트"
```

## 실패 조건

| 조건 | 동작 |
|------|------|
| `docs/05_설계서/` 없음 | 중단 → "/sl-recon 먼저 실행 필요" |
| 입력 파일 없음 | 중단 → "파일을 찾을 수 없음. 텍스트로 입력하거나 경로를 확인하세요" |

---

## STEP 1: 입력 파싱

입력이 파일 경로이면 Read한다. 텍스트이면 그대로 사용한다.

수집할 키워드:
- **화면/페이지 이름**: "화면", "페이지", "Form", "View", "List", "Detail" 인접 명사
- **기능 설명**: 동사+목적어 쌍 (예: "주문 취소", "배송지 변경", "할인율 조정")
- **데이터 항목명**: 컬럼명으로 보이는 것 (예: "주문상태", "배송일자", "할인코드")
- **SR-ID**: SR-YYYY-NNN 형식

---

## STEP 2: RECON 스펙 자동 매핑

추출한 키워드로 스펙 파일을 검색한다.

**INF 매핑:**
```bash
!grep -rl "{키워드}" docs/05_설계서/*/INF/ --include="*.md" 2>/dev/null
```

**UIS 매핑:**
```bash
!find docs/05_설계서/*/UI -name "spec.md" | xargs grep -l "{키워드}" 2>/dev/null
```

**SCH 매핑:**
```bash
!grep -rl "{키워드}" docs/05_설계서/ --include="DB_*.md" 2>/dev/null
```

키워드별로 매핑 결과를 수집하고 중복을 제거한다.

---

## STEP 3: 변경 규모 자동 분류

수집된 매핑 결과를 기준으로 분류한다:

| 분류 | 기준 |
|------|------|
| **Minor** | INF 1-2개 영향, SCH 변경 없음 |
| **Moderate** | INF 3개 이상 영향 OR SCH 컬럼 추가 예상 |
| **Major** | 신규 도메인 OR SCH 테이블 추가 예상 OR 아키텍처 변경 |

---

## STEP 4: 영향 리포트 생성

**SR-ID 결정:**
- 입력에 SR-YYYY-NNN 형식이 있으면 사용
- 없으면 오늘 날짜 기반으로 제안: `SR-{YYYY}-{NNN}` (docs/변경관리/ 기존 파일 수 기준)

**디렉토리 생성:**
```bash
!mkdir -p "docs/변경관리/{SR-ID}"
```

**리포트 저장:**
`docs/변경관리/{SR-ID}/00_영향분석_초안.md`

```markdown
# 영향 분석 초안 — {SR-ID}

**분석 일시**: {YYYY-MM-DD}  
**변경 유형**: 기능 수정 / 신규 기능 / 버그 수정  
**변경 규모**: Minor / Moderate / Major  

## 입력 내용 요약
{기획서/변경요청 핵심 내용 1-3줄}

## 영향 스펙

| 유형 | ID | 설명 | 예상 변경 |
|------|-----|------|---------|
| INF | {INF-ID} | {기능명} | {파라미터 추가 예상 / 응답 필드 추가 예상 / ...} |
| UIS | {UIS-ID} | {화면명} | {UI 변경 예상} |
| SCH | {없음 또는 SCH-ID} | {테이블명} | {컬럼 추가 예상} |

## 다음 단계

다음: /sl-analyze docs/변경관리/{SR-ID}/
```

**콘솔 출력:**
```
═══════════════════════════════════
기획서 분석 결과 — {SR-ID}
═══════════════════════════════════
변경 유형 : {분류}
변경 규모 : {Minor / Moderate / Major}

영향 스펙:
  INF  {INF-ID}  {기능명} — {예상 변경}
  UIS  {UIS-ID}  {화면명} — {예상 변경}
  SCH  {없음 또는 변경 내용}

저장: docs/변경관리/{SR-ID}/00_영향분석_초안.md

다음: /sl-analyze docs/변경관리/{SR-ID}/
═══════════════════════════════════
```
